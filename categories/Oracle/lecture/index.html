Day-19 Oracle Basic

day0719

  • SQL vs PL/SQL
  • SQL (분석가90% + 개발자 30%)
    • 프로그래밍 성격이 얕다
  • PL/SQL (분석가10% + 개발자70% +DBA)
  • DB분석
  • 사용자 설정 제약조건 확인

  • 테이블 정보 확인

제약조건

  • NOT NULL : 결측치 허용 X
  • UNIQUE : 중복값 허용 X
  • Primary Key
    • 기본키
    • UNIQUE, NOT NULL
    • 테이블당 1개의 기본키만 가능
  • 외래키
    • 테이블간의 참조 데이터 무결성 위한 제약 조건
    • 참조 무결성을 보장한다 -> 잘못된 정보가 입력되는 것을 방지
  • Check
    • 컬럼에 입력되는 데이터를 체크해 특정 조건에 맞는 데이터만 입력
  • Default
    • 값을 넣지 않아도 자동으로 입력

테이블 변경

  • 컬럼명 변경
  • 타입변경
  • 컬럼 추가
  • 컬럼 삭제
  • 제약조건 추가하기
  • 제약조건 삭제
  • 테이블 복사

뷰
