Day-21 Oracla Basic

기본 집계함수

  • count
  • DISTINCT
    • 종류를 추출

기초통계량

  • SUM
  • AVG
  • MIN,MAX
    • DISTINCT를 사용가능하지만 최소값과 최댓값을 똑같이 출력
  • VARIANCE, STDDEV
    • 분산과 표춘편차

GROUP BY절과 HAVING절

  • GROUP BY
    • 특정그룹으로 묶어 데이터를 집계할때 사용
    • WHERE와 ORDER BY절 사이에 위치
  • HAVING절
    • GROUP BY로 필터링된 결과를 대상으로 다시 필터를 건다
    • GROUP BY 다음에 위치
  • ROLLUP절
    • 명시한 표현식 수와 순서에 따라 레벨별로 집계한 결과가 반환
  • 분할 ROLLUP
  • CUBE
    • 명시한 표현식 개수에 따라 가능한 모든 조합별로 집계한 결과를 반환
    • 2^(expr의 수)
  • 분할 CUBE

집합 연산자

  • UNION
    • 합집합
    • 중복된 항목 한번만 표시
  • UNION ALL
    • 중복된 항목도 모두 표시
  • MINUS
    • 차집합
  • 집합 연산자의 제한 사항
    • 집합 연산자로 연결되는 각 SELECT문의 리스트 개수와 데이터 타입은 일치 해야한다
    • 집합 연산자로 SELECT 문을 연결할 때 ORDER BY는 맨 마지막 문장에서만 사용 가능
    • BLOB, CLOB, BFILE 타입의 컬럼에 대해서는 집합 연산자 사용 불가능
    • UNION, INTERSECT, MINUS 연산자는 LONG형 컬럼에 사용 불가능
  • GROUPING SET
    • ROLLUP이나 CUBE처럼 GROUP BY에 사용해서 그룹 쿼리에 사용
    • UNION ALL개념 포함

조인

  • 동등조인
    • 두 컬럼 값이 같은 행을 추출

  • 세미 조인

    • 서브 쿼리를 이용해 서브 쿼리에 존재하는 데이터만 메인 쿼리에서 추출
    • EXISTS 사용
    • IN 사용
      • 서브 쿼리 내 두 테이블의 조인 X

  • 안티 조인

    • 세미 조인의 반대 개념
    • NOT IN
    • NOT EXISTS

  • 셀프 조인

    • 테이블을 자기 자신과 연결
  • 외부 조인
    • 어느 테이블을 기준으로 하냐에 따라 데이터값이 다름
    • 외부조인은 조건에 해당하는 조인 조건 모두 (+)를 붙혀야한다
    • (+)연산자가 붙은 조건과 OR을 같이 사용할 수 없다
    • (+)연산자가 붙은 조건에는 IN연산자를 같이 사용할 수 없다
  • 카타시안 조인
    • 쓸 일이 없다

ANSI 조인

  • ANSI 내부조인
  • ANSI 외부조인
    • LEFT OUTER JOIN
    • RIGHT OUTER JOIN

서브쿼리

  • SELECT, FROM, WHERE
  • 연관성없는 서브쿼리
    • UPDATE, DELETE문 사용가능
  • 연관성있는 서브쿼리
  • 메인쿼리

  • MERGE

    • 조건을 비교 후 테이블에 해당 조건에 맞는 데이터가 없으면 추가
    • 있으면 UPDATE 수행

  • 테이블삭제

  • commit & rollback
    • commit 은 변경한 데이터를 데이터베이스에 마지막으로 반영
    • Rollback은 그 변경한 데이터를 변경하기 이전 상태로 되돌리는 역할
  • TRUNCATE TABLE
    • DELETE문과 같은 역활을 하는데 commit까지 되어서 Rollback이 안된다
  • ROWID, 주소 값
    • DBA, DB모델링 (쿼리속도 측정 –> 특징)

day0719

  • SQL (분석가90% + 개발자 30%)

제약조건

  • NOT NULL : 결측치 허용 X
  • UNIQUE : 중복값 허용 X
  • Primary Key
    • 기본키
    • UNIQUE, NOT NULL
    • 테이블당 1개의 기본키만 가능
  • 외래키
    • 테이블간의 참조 데이터 무결성 위한 제약 조건
    • 참조 무결성을 보장한다 -> 잘못된 정보가 입력되는 것을 방지
  • Check
    • 컬럼에 입력되는 데이터를 체크해 특정 조건에 맞는 데이터만 입력
  • Default
    • 값을 넣지 않아도 자동으로 입력
